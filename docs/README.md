# OctoAcme Project Management Docs

This README provides a high-level summary of OctoAcme's project management processes and quick links to the detailed process documents stored in this folder.

## Project Management Overview

OctoAcme follows an iterative, customer-focused project management approach designed to deliver value efficiently while maintaining clear accountability and quality standards.

### Key Principles

- **Iterative Approach**: Projects are executed in small, manageable increments with regular feedback loops
- **Customer-Focused**: Decisions are guided by customer needs and feedback
- **Clear Roles & Accountability**: Defined roles (PM, PdM, Developers, QA) ensure clear ownership
- **Lightweight Artifacts**: Practical documents (one-pagers, backlogs, risk registers) without unnecessary overhead
- **Regular Cadence**: Structured touchpoints (standups, weekly syncs, demos, retrospectives) keep teams aligned

### Project Lifecycle

OctoAcme's project lifecycle follows these key phases:

1. **Initiation**: Define project scope, goals, and approval with a one-pager and stakeholder buy-in
2. **Planning**: Build the backlog, estimate effort, and define definition of done (DoD)
3. **Execution & Tracking**: Execute work in small PRs with continuous integration and testing
4. **Release & Deployment**: Prepare for release with checklists, rollback plans, and communication
5. **Continuous Improvement**: Hold retrospectives, identify action items, and refine processes

### Team Structure

- **Project Manager (PM)**: Oversees project timeline, stakeholder communication, and risk management
- **Product Manager (PdM)**: Defines requirements, prioritizes backlog, and validates customer value
- **Developers**: Deliver code changes through small, tested PRs
- **QA**: Ensures quality through testing and compliance verification

### Communication & Cadence

- **Daily Standups**: 15-minute team syncs to discuss blockers and progress
- **Weekly Syncs**: Deeper reviews of progress, risks, and adjustments
- **Demos**: Regular showcases of completed work to stakeholders
- **Retrospectives**: Post-project reflections to capture lessons learned and improvements
